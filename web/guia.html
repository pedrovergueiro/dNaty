🧠 O que é o dNaty?

dNaty é um algoritmo de inteligência artificial criado por Pedro Vergueiro. Ele resolve três problemas que nenhum outro algoritmo resolve ao mesmo tempo:

  1. A rede neural muda de forma sozinha — ela cresce, encolhe e se reorganiza enquanto aprende
  2. Aprende com gradiente — usa matemática para melhorar os pesos da rede a cada passo
  3. Tem memória do que funcionou — lembra quais mudanças estruturais foram boas e repete elas
Imagine um estudante que não só estuda (gradiente), mas também reorganiza seus cadernos e métodos de estudo (estrutura), e anota quais técnicas funcionaram melhor para usar de novo (memória). O dNaty faz as três coisas ao mesmo tempo.

⚙️ Como o dNaty funciona por dentro?

A população de redes

O dNaty não treina uma rede — treina várias ao mesmo tempo (chamadas de "população"). Por padrão são 12 redes. Cada geração:

1
Cada rede passa por uma mutação estrutural — pode crescer, encolher, ganhar conexões novas, etc.
2
Cada rede treina com gradiente (Adam + cosine annealing) por alguns epochs
3
As melhores redes são selecionadas pelo NSGA-II (algoritmo de seleção multiobjetivo)
4
A memória episódica registra quais mutações melhoraram o modelo
5
Na próxima geração, mutações que funcionaram têm maior probabilidade de ser escolhidas

Os 10 operadores estruturais

São as "ferramentas" que o dNaty usa para mudar a estrutura da rede:

#NomeO que faz em linguagem simples
1add_neuronAdiciona um neurônio numa camada — a rede fica um pouco maior
2remove_neuronRemove o neurônio menos importante — a rede fica mais enxuta
3add_skipCria um atalho entre camadas (como no ResNet)
4change_activationTroca a função de ativação (relu → gelu, etc.)
5split_layerDivide uma camada em duas — mais capacidade
6merge_layersJunta duas camadas em uma — mais compacto
7prune_connectionsRemove conexões fracas — rede mais eficiente
8duplicate_moduleCopia uma camada com pequena variação
9add_conv_blockAdiciona bloco convolucional (para imagens)
10depthwise_sepAdiciona camada eficiente estilo MobileNet

A memória episódica

É como um caderno de anotações. Cada vez que uma mutação melhora a rede, o dNaty anota: "o operador X funcionou bem nessa situação". Com o tempo, o caderno fica cheio de experiências e o dNaty começa a escolher mutações mais inteligentes — não aleatórias.

📐 A matemática (versão leiga)

O Teorema dNaty-Convergence

O teorema principal prova que o dNaty sempre melhora a cada geração. Em linguagem matemática:

E[L_{g+1}] ≤ E[L_g] − δ_grad − δ_mem

Traduzindo:

  • L = o erro da rede (quanto menor, melhor)
  • g = número da geração
  • δ_grad = quanto o gradiente reduziu o erro (sempre positivo)
  • δ_mem = quanto a memória episódica ajudou (sempre positivo)

Isso significa: o erro na geração g+1 é sempre menor ou igual ao da geração g. O dNaty nunca piora — matematicamente provado.

O que é δ_grad?

É a melhora que vem do treino com gradiente. Quando você treina uma rede neural com Adam, os pesos se ajustam para reduzir o erro. Esse ajuste é o δ_grad. Quanto maior, mais o treino ajudou naquela geração.

Nos experimentos: δ_grad foi positivo em 100% das 355+ medições.

O que é δ_mem?

É a melhora que vem da memória episódica. Quando a memória guia uma mutação boa (que melhora a rede), isso contribui com δ_mem. Nas primeiras gerações é alto (memória aprendendo muito), depois estabiliza.

Nos experimentos: δ_mem foi positivo em ~95% das gerações.

🛠️ Como instalar e configurar

Pré-requisitos

Você precisa ter instalado:

  • Python 3.10 ou superior
  • pip (gerenciador de pacotes do Python)
  • Git (para clonar o repositório)

Passo a passo

1
Clone o repositório 
git clone https://github.com/pedrovergueiro/dNaty.git
cd dNaty
2
Instale as dependências 
pip install torch torchvision numpy scipy matplotlib tqdm scikit-learn
Se tiver GPU NVIDIA, instale a versão CUDA do PyTorch em pytorch.org/get-started
3
Verifique se funcionou 
python -c "from dnaty.operators.mutations import OPERATORS; print('OK:', OPERATORS)"

Deve aparecer: OK: ['add_neuron', 'remove_neuron', ...]

☁️ Google Colab (para CIFAR-10)

Por que usar o Colab?

O CIFAR-10 usa redes convolucionais (CNN) que são muito lentas no CPU — cada geração leva ~3 minutos. No Google Colab com GPU T4, leva ~30 segundos. O Colab é gratuito e oferece ~3h de GPU por dia.

1
Acesse colab.research.google.com
2
Ative a GPU: Runtime → Change runtime type → T4 GPU → Save
3
Faça upload do notebook: File → Upload notebook → selecione dnaty_colab.ipynb
4
Rode as células em ordem:
  • Célula 1 — verifica GPU
  • Célula 2 — instala dependências
  • Célula 3 — faz upload do dnaty_code.zip
  • Célula 8b — roda CIFAR-10 (~20 min)
  • Célula 10 — baixa o resultado
5
Copie o resultado para results/exp2_cifar10_results.json no seu projeto local

❓ Perguntas frequentes

Por que o dNaty é mais lento que um MLP normal?

Porque ele treina 12 redes ao mesmo tempo (população) e ainda faz mutações estruturais. Um MLP normal treina 1 rede. O dNaty compensa com resultados melhores e a capacidade de se adaptar continuamente.

O que é o FastDataset?

É uma otimização do dNaty v5. Em vez de carregar os dados do disco a cada epoch (lento), o FastDataset carrega tudo na RAM uma única vez. Resultado: 28× mais rápido que o DataLoader padrão do PyTorch.

# Exemplo de uso
from dnaty.experiments.fast_dataset import FastDataset
ds = FastDataset('MNIST', device='cpu')
# Agora ds.train_x está na RAM — zero I/O por geração
xb, yb = ds.get_train_batch(512)  # batch instantâneo

O que é BatchNorm e por que foi adicionado?

BatchNorm (Batch Normalization) é uma técnica que normaliza os valores dentro de cada camada durante o treino. Isso estabiliza o aprendizado e permite usar taxas de aprendizado maiores. Foi o principal responsável pelo salto de 90% para 98.70% no MNIST.

O que é early stopping?

É quando o dNaty para de rodar gerações antes do limite máximo porque percebeu que não está mais melhorando. Por padrão, para se não melhorar em 8 gerações consecutivas. Isso economiza tempo sem perder qualidade.

Como interpretar o δ_grad e δ_mem no terminal?

Quando você roda o dNaty, aparece algo como:

Gen  5 | acc=0.9752 | δ_grad=0.31406 | δ_mem=0.00439 | params=56810
  • acc = acurácia atual (0.9752 = 97.52%)
  • δ_grad = quanto o gradiente ajudou nessa geração (maior = melhor)
  • δ_mem = quanto a memória episódica ajudou (maior = melhor)
  • params = número de parâmetros da melhor rede

Posso usar o dNaty nos meus próprios dados?

Sim. Para dados tabulares (planilhas, CSVs), crie um TabularFast similar ao do run_real_world.py. Para imagens, use o FastDataset com CIFAR ou crie um similar. O algoritmo em si não muda — só o carregamento dos dados.
